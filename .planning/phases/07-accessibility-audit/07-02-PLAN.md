**WCAG 2.1 AA Criteria for Forms:**

1. **Labels (1.3.1, 3.3.2)**
   - Programmatic label association
   - Visible labels
   - Instructions provided

2. **Error Handling (3.3.1, 3.3.3)**
   - Error identification
   - Error suggestion
   - aria-invalid, aria-describedby for errors

3. **Input Purpose (1.3.5)**
   - autocomplete attributes where applicable

4. **Focus (2.4.7)**
   - Visible focus indicators

**WCAG 2.1 AA Criteria for Navigation:**

1. **Multiple Ways (2.4.5)**
   - Consistent navigation patterns

2. **Headings and Labels (2.4.6)**
   - Descriptive navigation labels

3. **Link Purpose (2.4.4)**
   - Link purpose clear from text or context

**WCAG 2.1 AA Criteria for Feedback:**

1. **Status Messages (4.1.3)**
   - role="status" or role="alert" for dynamic content
   - aria-live regions for updates

**Components:** Input, Select, Checkbox, Toggle, Textarea, DatePicker, Form, FormBuilder

**Audit checklist per form control:**
- [ ] Label association (htmlFor/id or aria-labelledby)
- [ ] Required field indication (aria-required, visual indicator)
- [ ] Error state handling (aria-invalid, aria-describedby)
- [ ] Help text association (aria-describedby)
- [ ] Disabled state (aria-disabled or disabled attribute)
- [ ] Focus indicator visibility
- [ ] Keyboard operability
- [ ] Touch target size (minimum 24x24px for WCAG 2.1)

**For Select/DatePicker specifically:**
- [ ] Custom widget keyboard navigation
- [ ] ARIA combobox/listbox pattern if custom
- [ ] Native fallback consideration

**For Checkbox/Toggle:**
- [ ] aria-checked state (if custom)
- [ ] Group labeling (fieldset/legend or aria-labelledby)

Document Current State, Gaps, and Priority for each.

- [ ] nav with aria-label="Breadcrumb"

- [ ] Current page indication
- [ ] Disabled state for unavailable pages
- [ ] Page count announcement

**Feedback components:** Alert, Toast, Progress, Spinner, Skeleton

**Feedback checklist:**
- [ ] role="alert" for important messages
- [ ] role="status" for status updates
- [ ] aria-live for dynamic content
- [ ] aria-busy for loading states
- [ ] Progress: aria-valuenow, aria-valuemin, aria-valuemax
- [ ] Skeleton: aria-hidden when decorative, proper loading announcement
